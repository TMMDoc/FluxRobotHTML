Robot Coordinates

Joint coordinate system

Joint coordinate system The joint coordinate system affects the robot joints. The position and behavior of the robot are defined by angle shifts in relation to the joint coordinate system of the joint base.

Fanuc specific Axis calculation

Important
 In Fanuc the motion of the J3 axis is dependent on the motion of J2: J3=J3+J2

Jogging the J3 axis keeps the arm e.g. horizontal. A motion of J3 does not affect J2

World coordinate system

This standard coordinate system is fixed for the robot in the intersection of the J1 and J2 axis. Z is up, X is along the base (Blue)

User coordinate system

Real components of your system slightly deviate from the digital representation of it in FluxRobot. It is advised to keep Fluxrobot as accurate as possible to get the most accurate program possible (automated collision check !) but slight corrections could be made on the machine without reprogramming in the offline system again.

User coordinate systems could be experimentally measured with the Fanuc 3 point measuring. See Fanuc manual for detials. Below an example is illustrated:

The following example shows how to get the angles from a fanuc 3 point measurement The result is a userframe [14 0 42]

Fanuc uses the coordinates W P R which correspondent to → Rotation X Y Z The meaning of (W, P, R) is shown below.

Xu, Yu and Zu: The coordinate system is defined in the working range.
Xt, Yt and Z: The coordinate system is defined for the tool
NOTE: The User Frames are usually defined 50mm inside the 1mm strong calibration plate

Tool coordinate system

The tool coordinate system allows the user to jog the robot in the direction of the gripper. The standard orientation of the tool coordinate system is chosen that the orientation of the tool alligns with the machine coordinate system within the standard bending process.

Note
 The TCP is the defined by the 1mm calibration plate under vaccuum.

Jog coordinate system (JGFRM)

Jog frames help to move the Robot aligned with components. FluxRobot defines the individually along the motions, e.g. on bendig lines. If you need to readjust your part it might help to rotate he part and extract it with addtional bending points.
